# Analyzing Session Patterns

セッションを分析し、**スキルの使用状況と改善機会**を特定する。既存スキルの効果測定、改善提案、新規スキルの必要性判断を行う。

## When to Use

Use this skill when:
- 「このセッションを振り返って」
- 「振り返りをして」
- 「セッションを分析して」
- 「改善点ある？」
- 「スキルの効果を確認して」

## Workflow Overview

このスキルは7つのステップで構成されています：

1. **既存スキルの読み込み**: 利用可能なスキルの全体像を把握
2. **スキル使用状況の分析**: 使用されたスキル、使用されなかったスキル、手動作業を評価
3. **並列化の分析**: サブエージェント（Task）の使用状況と効率を評価
4. **エラー・ミスパターンの分析**: 繰り返しのエラーとユーザー修正指示を特定
5. **改善提案の作成**: 優先度順（高：スキル改善、中：新規スキル、低：CLAUDE.md追記）
6. **レポート出力**: 包括的な分析レポートを生成
7. **ユーザーへの確認**: 改善の実行方針を確認

## Analysis Focus

### スキル使用状況
- ✅ **使用されたスキル**: トリガー、ワークフロー、期待値との一致を評価
- ⚠️ **使用機会を逃したスキル**: トリガーの改善が必要
- 📝 **手動で行った作業**: スキル化の候補を特定

### 並列化の評価
- **適切な並列化**: 独立したタスクを効率的に並列実行
- **機会損失**: 並列化すべきだった場面
- **不適切な並列化**: 順序依存や不要な委譲

### 改善提案の優先度
1. **高**: 既存スキルの改善（トリガー、ワークフロー）
2. **中**: 新規スキルの提案
3. **低**: CLAUDE.md追記

## Notes
